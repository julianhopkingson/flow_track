# 图标颜色重设计方案 v1.0

作为首席架构师，针对将“显示桌面”、“语言”和“文件夹”图标颜色统一修改为绿色的需求，本方案旨在提供一个既能满足视觉统一性，又能保持系统逻辑一致性的具体实现路径。

## 设计目标
- **视觉一致性**：将三个特定图标替换为与系统中其他功能性图标（如点击数、间隔）一致的绿色（十六进制码：`#26D07C`）。
- **功能稳定性**：仅修改视觉表现（Pixmap 渲染颜色），严禁触动底层业务逻辑。
- **状态感知**：保留原有的“激活/禁用”色彩状态逻辑，即当 UI 锁定或功能不可用时，图标仍需能够正确变回灰色（`#CBD5E0`）。

## 技术方案说明

### 1. 颜色标准
方案建议统一使用系统中已有的品牌绿：
- **目标绿色**：`#26D07C` (品牌一致性推荐)
- **备选绿色**：`#10B981` (如果需要更高的对比度)
- **禁用灰色**：`#CBD5E0` (保持现有逻辑)

### 2. 修改范围
| 图标名称 | 所在位置 | 对应文件 | 当前颜色 | 目标颜色 |
| :--- | :--- | :--- | :--- | :--- |
| **语言 (Globe)** | 主窗口标题栏 | `ui/main_window.py` | `#4A90E2` (蓝) | `#26D07C` |
| **文件夹 (Folder)** | 加载配置按钮 | `ui/main_window.py` | `#ECC94B` (黄) | `#26D07C` |
| **显示桌面 (Desktop)** | 定时器卡片内项 | `ui/components/timer_card.py` | `#3182CE` (蓝) | `#26D07C` |

### 3. 实现细节解析
- 在 `ui/main_window.py` 的 `update_header_icons` 方法中同步更新变量定义。
- 在 `ui/components/timer_card.py` 的 `update_icon_states` 方法及初始化块中同步更新硬编码的颜色字符串。

## 潜在风险评估

1. **视觉层次缺失**：
   - *风险*：如果界面中过多的图标统一为绿色，可能会降低各个功能组之间的视觉区分度，用户在扫视界面时定位特定功能的难度可能略微增加。
   - *对策*：通过图标形状差异和布局分组来弥补颜色单一带来的识别困难。

2. **对比度问题**：
   - *风险*：目前的背景为浅色/透明质感，`#26D07C` 在某些光照环境或特定背景色下可能不如深蓝色清晰。
   - *对策*：已确认该绿色在当前主题下具备足够的易读性。

3. **状态遗漏**：
   - *风险*：若仅修改初始化颜色而遗漏了动态切换状态的代码（如 `on_desktop_toggled`），会导致图标在功能切换后“变回”旧的颜色。
   - *对策*：方案已覆盖所有涉及该图标重绘的函数入口（`init_ui` 和 `update_icon_states`）。

## 后续步骤
1. 用户输入“**执行**”。
2. 系统将按照上述方案逐个更新文件。
3. 进入验证阶段，确保图标在正常态、禁用态及多语言切换时表现正确。
