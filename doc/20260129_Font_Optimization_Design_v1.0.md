## 2. 问题分析与优化目标

### 日志字体问题 (图2)
根据您提供的截图，目前的日志字体在混合中英文时不够平衡：
- **等宽性冲突**：`Consolas` 是优秀的西文等宽字体，但在 Windows 下通常无法完美匹配中文字体的基线和宽度，导致排版略显凌乱。
- **美观度不足**：在界面追求 "Rich Aesthetics" (丰富美感) 的背景下，默认的等宽字体显得过于硬核，不够优雅。

### 优化目标
- **增强日志可读性**：采用更高颜值的现代等宽或混合字体方案。
- **视觉一致性**：确保日志部分既能保持代码/日志感，又能与全局的现代感设计融合。

---

## 3. 详细设计解决方案

### 方案 A：引入现代字体栈 (推荐)
这是目前生产级软件中最平衡的方案。

- **日志字体建议**：`'JetBrains Mono', 'Cascadia Code', 'Segoe UI', 'PingFang SC', 'Microsoft YaHei', monospace`
- **实现建议**：
  - 增加行间距 (`line-height`) 到 1.4 或 1.5。
  - 略微调整字体大小为 9.5pt 或 10pt。

### 方案 B：统一全局非等宽字体 (极简风格)
如果日志不需要强调 "代码感"，可以完全使用全局字体。
- **字体建议**：`'Segoe UI Semibold', 'Microsoft YaHei UI', sans-serif`

---

## 4. 潜在风险与应对方案

1. **字体缺失风险**：如果用户系统未安装特定的 Monospace 字体。
   - **对策**：在 QSS 列表中保留 `Consolas` 和 `微软雅黑` 作为兜底 (Fallback)，确保 UI 不会乱码。
2. **行高自适应**：QTextEdit 默认行高较为紧凑。
   - **对策**：通过 QSS 或内联样式微调，增加空气感。
